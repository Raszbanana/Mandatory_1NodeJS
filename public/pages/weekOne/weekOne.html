Designing a REST API




One of the first task's was to design a REST API. Most importantly doing so in a correct manner, following the right conventions. We decided to have memes as our entity. Above is how i went about designing the API

The important takes i gathered was:


  • We can identify the “memes” collection resource using the URI “/memes“. We can identify a single “meme” resource using the URI “/memes/{memeId}“. Also referred to as the pathvariable

  • If the searched ressource is a multiple worded query, utilize "-" to connect the words into a single word.
    A good example would be: meme manager > /meme-managers > /meme-managers/{meme-managerId}

    • Basic terminal commands

  • $ cd "insertDirectory"

    cd into a directory, which is within the direct scope of your current directory


  • $ cd ..

    Move up on directory


  • $ ls OR dir

    View the files inside your current scope


  • $ mkdir "nameOfDirectory"

    Create a new directory inside your current one

    • Variables & Identifiers

    $ let amount;

    Here we declare a "let" variable, which we identify by the keyword "amount"


    $ amount = 5;

    Here is where we initialize the variable identified by the keyword "amount", to be equal to the number 5
