About functions

Important function types are:


$ function AddArrowFunction(x,y) { return x+y } // Named

$ const addArrowFunction = function(x,y) { x+y; } // Anonymous

$ const addArrowFunction = (x,y) => { x+y;} // Arrow or Lambda

Assigning a variable to a function, still makes the function anonymous


Currying & Callbacks

Are two completely different technologies. Callbacks are essentially a synonym for "passing a function to a function" (i.e. higher-order function that consumes a function). In essense, currying is simply the act of returning a usable function in JavaScript instead of a primitive value like a Boolean or a string:


$ function functionalThing(noun) { return phrase => console.log(`I am ${noun}, and ${phrase}.`);} // Currying

Code Conventions, why?

Code conventions improve the readability of the software, allowing engineers to understand new code more quickly and thoroughly. If you ship your source code as a product, you need to make sure it is as well packaged and clean as any other product you create.

Make sure you code always follows in line with the general convention of whatever you are programming. The general rule of thumbs for programming in general is:


"lowerCamelCase when naming constants, variables, and functions. Use UpperCamelCase (capital first letter as well) when naming classes. This will help you to easily distinguish between plain variables/functions, and classes that require instantiation. Use descriptive names, but try to keep them short."


Conventions can basically range from file organization, indentation, comments, declarations, statements, white space, naming conventions, programming practices, programming principles, programming rules of thumb to architectural best practices, etc

NodeJS with Express

Here is how you setup a simple and easy server instance, using express. Assuming you have already installed NPM you now need to install express:


$ npm install express --save

Once this is done you can now create your app.js file and type in the following:


$ const express = require("express");

This is basically how you import in node, using require instead. We are assigning the const express to the imported contents of the express module


Once this is done we need to assign our const "app" to the express() function. This is basically what instantiates a new express application


$ const app = express();

Now that we have setup express, we can now start to use the built-in functions to take requests


$ app.get('/', (req, res) => { res.send('Hello World!') })

However, since we didn't specify a port yet, we also need to assign that so that we know where to listen for requests


$ app.listen(3000, (req, res) => { console.log("App listening on port 3000"); });

Now when we navigate to "localhost:3000" We will be returned by a text that says hello world. Like this:


"Hello World!"
